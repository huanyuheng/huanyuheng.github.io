## 三、如何修改 / 更新 GitHub 代码

> 回答“你可以直接改 GitHub 里的代码吗？”：我们会在本地/云端工作目录中改完再推送到 GitHub，GitHub Pages 随后会自动重新构建站点。

### 协作修改的标准流程
- **获取代码**：从 GitHub 仓库拉取最新内容（`git clone` 或 `git pull`）。
- **本地编辑**：在工作目录内新增/修改文章、页面或样式，遵循上面的目录结构和 Jekyll 前置字段格式。
- **自测预览（可选）**：安装 Jekyll 依赖后运行 `bundle exec jekyll serve`，在本地预览生成效果。
- **提交与推送**：执行 `git add`、`git commit`、`git push`，将修改同步到 GitHub。
- **自动发布**：仓库启用 GitHub Pages 后，推送完成会自动触发构建，几分钟内即可在自定义域名上看到最新内容。

### 需要注意
- 直接在 GitHub 网页端在线编辑也可以，但更推荐本地改完再一次性提交，方便备份和回滚。
- 提交前检查 `_posts` 日期格式、Front Matter 以及 `assets/css/style.scss` 等静态资源是否符合 Jekyll 要求，以免构建失败。
- 如果涉及第三方脚本或资源路径，记得使用相对路径，确保在 Pages 构建后能正常加载。

### 如果必须在 GitHub 网页端快速修改
1. 进入目标文件，点击右上角的铅笔图标进入编辑模式。
2. 修改后在底部写清提交说明，选择“Create a new branch for this commit and start a pull request”更安全。
3. 点击“Propose changes”，再在 PR 页面完成“Create pull request”，等待自动构建完成。
4. 如需直接提交到默认分支，务必确认改动无语法问题，避免阻塞 Pages 构建。
